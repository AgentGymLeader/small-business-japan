# Customer Pulse JP

## Quick start

「今月、顧客はどんな反応をしているか？」と聞いてみる。

Stripe の返金・チャージバック、サポートメール、貼り付けたレビューを集計し、
3〜5テーマを証拠付きでまとめ、「今週やること3つ」リストを作成する。

Google/食べログ/Googleマップのレビューを含める場合は、
トリガー後に貼り付けるか「レビューを追加したい」と言う。

## Workflow

1. **対象期間を設定する。** デフォルトは過去30日。オーナーが期間を指定した場合はそれに従う。

2. **Stripe の返金・チャージバックを取得する。** 対象期間内に発生した返金・チャージバックを取得する。APIがレート制限に達した場合はスキップし、Sources セクションに「Stripe: レート制限によりスキップ」と記載する。

3. **サポートメール・チケットを取得する。** Gmail または接続されたサポートツール（Zendesk等）から以下のキーワードを含むメールを検索する：「返金」「キャンセル」「不満」「問題」「遅れ」「壊れている」「間違っている」「届いていない」。件名と本文抜粋（1〜2文）を取得する。

4. **貼り付けられたレビューを受け付ける（任意）。** Google マップ、食べログ、その他のレビューサイトからのレビューテキストを貼り付けてもらえる場合は含める。タグは `[レビュー]` とする。

5. **テーマを抽出する。** 全証拠を3〜5のテーマにグループ化する。各テーマに含める：
   - 1文のラベル（例: 「配送遅延に関する繰り返しのクレーム」）
   - 原文引用2〜3件（ソースタグ付き: `[Stripe]`、`[Gmail]`、`[レビュー]`）
   - シグナル件数（このテーマに関連する件数）

   原文引用は必須——要約で代替しない。

6. **「今週やること3つ」リストを作成する。** テーマをシグナル件数で順位付けし、上位3テーマについてオーナーが実行できる具体的な対応を1つずつ作成する。

7. **レポートを出力する。** 以下の順で構成する：
   - **ヘッダー** — 「顧客の声レポート」と対象期間
   - **取得ソース** — ソース別のシグナル件数（レート制限でスキップしたものも記載）
   - **テーマ** — 各テーマに太字のラベル・シグナル件数、原文引用2件（ブロック引用）
   - **今週やること3つ** — 上位テーマに紐づいた3つの具体的なアクション

   **個人情報・消費者法対応チェック（出力時に注意）**:
   - 集計テーマには個人を特定できる情報を含めない（氏名・連絡先等）
   - 個人情報の利用目的外使用に該当する分析は行わない（個人情報保護法第18条）
   - 通信販売の顧客クレームは特定商取引法上の広告記載義務違反の可能性もある——法的判断は不要だが「要確認」フラグを立てる

## Approval gates

このスキルは **読み取り専用** — 投稿・送信・返信・レコードの修正は行わない。

- **顧客クレームの法的責任・返金適否を確定的に判断しない。**
- **個人情報保護法（APPI）に基づく開示・訂正・削除・利用停止請求への対応が完了したとは言わない。**
- **集計レポートに個人を特定できる情報（氏名・メールアドレス等）を含めない。**（個人情報保護法 利用目的の特定・最小化原則）
- **このレポートから送信可能な返信メールを生成しない。** 別途 `ticket-deflector-jp` のドラフト承認フロー経由で行う。

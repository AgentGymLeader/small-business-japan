# Customer Pulse Check JP

顧客の声を集約して分析する。接続されているすべてのソースからフィードバックシグナルを取得し、実際に修正できる課題を特定して、オーナーが確認・送信できる返信下書きを作成する。

引数を解析する：
- `--since`（デフォルト: 過去30日）— 集計期間の開始日 `YYYY-MM-DD`

## Step 1 — フィードバックシグナルの収集

`customer-pulse-jp` スキルのワークフローを使用する：

1. 対象期間の Stripe の返金・チャージバック（理由コード・金額・解決状況）を取得する。
2. 対象期間のサポートメール・問い合わせ（Gmail等）を取得する。
3. レビューエクスポートファイル（Google レビュー CSV 等）がある場合は読み込んで解析する。
4. ソース別のシグナル件数を集計する。

## Step 2 — テーマ抽出

全シグナルを繰り返し発生するテーマにグループ化する。各テーマに：
- テーマに言及するシグナル件数
- 分類: 商品品質 / 配送 / 請求 / コミュニケーション / 期待値のズレ / その他
- 影響度評価: 🔴 高（収益リスク・解約） / 🟡 中 / 🟢 低

## Step 3 — 修正可能な上位3課題

`ticket-deflector-jp` スキルのワークフローを使用する：

件数 × 影響度評価で上位3テーマを選択する。各テーマについて：
1. 課題を1文で説明する
2. 根本原因を説明する（判明している場合）
3. 具体的な業務改善案を提案する
4. 顧客返信テンプレートを下書きする

返信テンプレートの形式：
```
件名: Re: {課題のトピック}

{氏名} 様

この度はご連絡をいただきありがとうございます。{1〜2文で状況への共感}.

{対応内容・発生した事情・提供する解決策}.

{次のステップまたは提案}.

{署名}
```

## Step 4 — サマリー表

以下の形式で出力する：

```
顧客の声 — {期間}
シグナル合計: {n}件（Stripe返金: {n}件 | サポートメール: {n}件 | レビュー: {n}件）

修正可能な上位3課題
1. {課題}（{件数}件）— {影響度} — 改善案: {1行}
2. {課題}（{件数}件）— {影響度} — 改善案: {1行}
3. {課題}（{件数}件）— {影響度} — 改善案: {1行}
```

## コネクター障害時の対応

接続されているソースで実行する——このコマンドはグレースフルにデグレードする。Stripe が接続されていない場合は返金データをスキップして「Stripe未接続——返金データなし」と注記する。ソースがまったく接続されていない場合は停止して「フィードバックソースが接続されていません。Stripe、Gmail、またはレビューCSVのいずれかを接続してください」と伝える。

## Approval gates

- **返信メールを自動送信しない。** オーナー確認用の下書きのみ提示する。
- **返金・チャージバックをオーナーの明示的な承認なしに解決・処理しない。**
- **顧客の個人情報をサマリーに含めない** — 姓のみを使用する。

## Output

サマリー表を提示し、続いて各返信テンプレートを表示する。どのテンプレートを送信するかオーナーに確認し、明示的な承認を待つ。

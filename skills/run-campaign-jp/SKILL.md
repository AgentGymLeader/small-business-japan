# Run Campaign JP

3つのスキルを順番に連携させてフルキャンペーンを実行する。
各ステップでオーナーの承認を得る——承認なしに次のステップへは進まない。

引数を解析する：
- `--lookback`（デフォルト: `90d`）— 売上ダウンの調査期間
- `--channel`（デフォルト: `both`）— `email`、`sns`、または `both`

## Step 1 — 売上分析とコンテンツブリーフ（content-strategy-jp）

`content-strategy-jp` スキルを実行する：
1. freee と Stripe の売上データを指定期間で取得する。
2. 売上ダウンの特定——どの商品・サービス、いつ、どの程度。
3. 30日間のコンテンツブリーフ作成：何を押すか、どんなプロモをするか、何を止めるか。
4. ブリーフをオーナーに提示する。「承認、素材作成へ」を受けるまで次へ進まない。

オーナーがブリーフを修正した場合は反映して再提示する。

## Step 2 — 素材作成とステージング（design-tool-handoff-jp）

Step 1 承認後、`design-tool-handoff-jp` スキルを実行する：
1. 承認済みブリーフを入力として受け取る。
2. ブリーフの優先度に合わせた投稿カレンダーを作成する。
3. 各投稿のキャプション案を作成する（Canva等での素材作成も提案する）。
4. オーナーに提示する。「承認、リスト作成へ」を受けるまで次へ進まない。

## Step 3 — ターゲットリストの作成（lead-triage-jp）

Step 2 承認後、`lead-triage-jp` スキルを実行する：
1. 承認済みブリーフのターゲットセグメントに合うリードを kintone・スプレッドシートから取得する。
2. エンゲージメント・適合性・緊急度でスコアリングする。
3. 2つの成果物を作成する：
   - **一斉送信リスト** — キャンペーンのターゲットセグメント
   - **重点連絡リスト** — オーナーが直接電話すべき上位5社とアプローチポイント
4. 両リストを提示する。「送信」コマンドを受けるまでキャンペーンを実行しない。

## Approval gates（必須）

- **ステップ間を自動で進めない。** 各引き渡しにはオーナーの明示的な承認が必要。
- **キャンペーンの送信はオーナーの「送信」コマンドなしに実行しない。**
- **コネクター（freee、Stripe、Canva等）に接続できない場合は停止し、何が失敗したかを報告してリトライか中止かを確認する。**

## Output

最後に1段落でサマリーを出力する：特定した課題、作成した投稿数、リストのセグメントサイズ、重点連絡社数。

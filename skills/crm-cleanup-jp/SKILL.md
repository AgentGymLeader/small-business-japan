# CRM Cleanup JP

## Quick start

`crm-maintenance-jp` で作成した整理計画をもとに、
オーナーが承認した修正を実行するスキル。各変更前に確認を求める。

## Workflow

1. **整理対象を確認する。** オーナーがどの整理作業を実行したいか確認する：
   - 重複のマージ
   - 情報不足レコードの補完
   - 活動停止顧客のステータス変更
   - フォローアップ日付の更新

2. **重複のマージ（承認が必要）。** `crm-maintenance-jp` で検出した重複ペアを1件ずつ提示する。どちらを基準レコードにするかオーナーが選択する。承認後にマージを実行する。

3. **情報の補完（オーナー入力が必要）。** 情報不足のレコードについて、不足している情報をオーナーに質問する。回答を受けてからレコードを更新する。

4. **活動停止顧客の対応（オーナーが判断）。** 活動停止の顧客リストを提示し、各顧客についてどう対応するかオーナーが選ぶ：
   - フォローアップメールを下書きする
   - 「休眠」ステータスに変更する
   - 削除する（この場合は確認を2回求める）

5. **変更ログを作成する。** 実行したすべての変更を記録し、レポートとして提示する。

## Approval gates

- **オーナーの明示的な承認なしに顧客データを変更・削除しない。**
- **一括処理を行わない。** 1件ずつ確認を取る。
- **削除はオーナーの明示的な承認なしに実行しない。** 削除前に二重確認を行う。
- **変更後に変更ログを必ず提示する。**

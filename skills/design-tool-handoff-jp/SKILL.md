# Design Tool Handoff JP

## Quick start

コンテンツブリーフ（または口頭での要望）をもとに、
Canva 等のデザインツールで素材を作るための指示書を作成する。
デザインファイルの直接編集・投稿は行わない。

```
オーナー: 「このブリーフでSNS素材を作って」
→ ブリーフの内容確認
→ 投稿カレンダーの作成
→ 各投稿のデザイン指示書作成
→ キャプション案の作成
→ オーナー確認
```

## Workflow

1. **入力を確認する。** 以下のいずれかを受け取る：
   - `content-strategy-jp` または `sales-brief-jp` で作成した承認済みブリーフ
   - オーナーからの直接の要望（「週末のセール告知を作りたい」等）

   承認済みブリーフがない場合は、「何のために・どんな内容の・いつ投稿する素材ですか？」を確認する。

2. **投稿カレンダーを作成する。** ブリーフの優先度に合わせた2週間の投稿カレンダーを作成する：
   - 投稿日・曜日
   - 投稿先チャネル（Instagram / X / LINE / Facebook 等）
   - 投稿の目的（認知拡大・商品PR・お知らせ等）

3. **各投稿のデザイン指示書を作成する。** Canva 等で作業する人（またはオーナー自身）に向けた指示書：

   ```
   投稿 #{番号} — {日付}（{曜日}）
   チャネル: {SNS名}
   サイズ: {1080×1080px / 1080×1920px 等}

   【メインビジュアル】
   {背景色・メイン画像のイメージ}

   【テキスト】
   メイン: "{コピー}"
   サブ: "{サブコピー}"

   【ブランドカラー・フォント】
   （オーナーのブランドに合わせて設定）

   【雰囲気・参考】
   {明るい・信頼感・親しみやすい等}
   ```

4. **キャプション案を作成する。** 各投稿のSNSキャプションを下書きする：
   - 本文（投稿の目的に合わせたトーン）
   - ハッシュタグ（業種・内容に合わせた日本語・英語ハッシュタグ）
   - CTA（コールトゥアクション: 「詳しくはプロフィールから」等）

5. **オーナーに確認する。** 投稿カレンダーとデザイン指示書・キャプション案を提示し、
   修正があれば対応する。

6. **Canva 等への引き渡し準備をする。** 以下をまとめた引き渡しパッケージを作成する：
   - 投稿カレンダー（CSV等で貼り付け可能な形式）
   - 各投稿のデザイン指示書
   - キャプション一覧

## Approval gates

- **SNS への自動投稿・スケジュール設定は行わない。**
- **Canva 等のデザインツールへの直接アクセス・編集は行わない。**
- **投稿内容はオーナーの明示的な承認なしに公開・送信しない。**
- **外部サービスへの画像アップロードは行わない。**

# Lead Triage JP

## Quick start

リード情報をスコアリングし、優先度の高い見込み客への
アプローチ文下書きを作成する。送信・登録は行わない。

```
オーナー: 「営業連絡リストを作って」
→ リードデータの取得
→ 4軸スコアリング（エンゲージメント・適合性・緊急度・最終接触日）
→ 優先度順リストの作成
→ アプローチ文の下書き作成
```

## Workflow

1. **リードデータを取得する。** kintone、スプレッドシート（顧客管理シート）、または Salesforce からリードを取得する。接続されていない場合は、リスト（CSV等）を貼り付けてもらう。

2. **トリガーの意図を確認する。** 「パイプライン」「リスト」など曖昧な表現の場合は確認する：
   「パイプラインの概要（商談状況と合計金額）ですか？それとも優先連絡リストですか？」

3. **スコアリングを行う。** 4軸で評価する：
   - **エンゲージメント** — 過去30日のメール返信・資料開封・商談履歴
   - **適合性** — 業種・規模がターゲットと一致するか（デフォルト: 従業員1〜50名）
   - **緊急度** — 接触からの経過日数、「検討中」「予算あり」「今期中に」等のキーワード
   - **最終接触ペナルティ** — 24時間以内に接触済みの場合は点数を下げる

4. **優先度順リストを作成する。** スコア降順でリストを作成する：
   - リード10件以下 → 全件表示
   - 11〜30件 → 上位5件
   - 30件超 → 上位8件

   各リードに: 会社名、担当者名、スコア、アプローチポイント（1段落）、最終接触日を表示。
   エンゲージメントシグナルが30日以上前の場合は「コールドアプローチとして対応を」と注記する。

5. **アプローチ文の下書きを作成する。** 「この中でアプローチ文を作りますか？」と確認し、
   選択されたリードについてメール下書きを作成する。過去の送信メールのトーンに合わせる。
   下書きのみ提示する——送信しない。

6. **商談化の提案を行う。** 特定のリードについてカレンダー日程を提案する場合は、
   「候補日を提示しますか？」と確認してから実行する。実際のカレンダー登録はしない。

## Approval gates

- **メールの送信は行わない。** 下書きのみ提示する。
- **カレンダーへの登録は行わない。** 候補日の提示のみ。
- **リードのステータス変更は行わない。** 読み取り専用。
- **個人情報（氏名・連絡先）をログや外部に出力しない。**
- オーナーの明示的な承認なしに、メール送信・CRMへの書き込み・ステータス変更・カレンダー登録は行わない。

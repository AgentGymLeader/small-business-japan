# Price Check JP

## Quick start

売上・原価データから現在の価格設定を評価し、
価格変更シミュレーションと改善案を提示する。
価格の変更・システムへの反映は行わない。

```
オーナー: 「価格確認をして」
→ 現在の価格・コスト構造の確認
→ 価格変更シミュレーション
→ 競合・市場との比較（可能な場合）
→ 価格戦略の改善案提示
```

## Workflow

1. **現在の価格・コスト構造を確認する。** freee または マネーフォワードクラウドのデータから：
   - 商品・サービス別の現在の販売価格
   - 対応する原価・直接費用
   - 現在の粗利率

   データがない場合は、オーナーに価格と原価を入力してもらう。

2. **価格変更シミュレーションを行う。** オーナーが価格変更を検討している場合：
   - 現在: 価格 {X}円、原価 {Y}円 → 粗利率 {Z}%
   - 変更案: 価格 {X+α}円 → 粗利率 {Z+β}%
   - 同じ粗利額を達成するために必要な販売数量の変化

3. **損益分岐点を計算する。** 固定費が把握できている場合：
   - 現在の損益分岐点（月間販売数 / 売上金額）
   - 価格変更後の損益分岐点

4. **消費税の影響を確認する。** 価格変更の際に消費税（8% または 10%）の取り扱いを確認する：
   - 税込み価格 vs 税抜き価格の表示
   - インボイス制度下での適格請求書の発行

5. **価格戦略の改善案を提示する。** 以下のパターンで提案する：
   - 粗利率が業種平均より低い場合: 「価格見直しまたは原価削減を検討してください」
   - 競合との比較で割安な場合: 「値上げ余地があります（業種平均: {%}）」
   - 価格幅を広げることで対応できる場合: 「スタンダード / プレミアムプランへの分離を検討してください」

6. **レポートを作成する。** 以下の形式でまとめる：

   ```
   価格確認レポート — {日付}

   【現在の価格・収益性】
   商品・サービス名 | 価格 | 原価 | 粗利率
   {商品A}        | {円} | {円} | {%}
   {商品B}        | {円} | {円} | {%}

   【価格変更シミュレーション】
   {変更案の内容と影響}

   【改善提案】
   ・{提案}
   ・{提案}

   ※ 最終的な価格決定はオーナーにて判断してください。
     税務上の影響については税理士にご確認ください。
   ```

## Approval gates

- **価格の変更・システムへの反映は行わない。** 分析・提案のみ。
- **「この価格が正しい」「儲かります」等の保証はしない。**
- **消費税・インボイス制度の最終判断は税理士に確認するよう促す。**
- **オーナーの明示的な承認なしにStripe・STORESの価格設定を変更しない。**

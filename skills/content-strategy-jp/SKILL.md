# Content Strategy JP

## Quick start

売上データを分析して売れ筋と季節性を把握し、
「何を押すか・何を止めるか」の30日間コンテンツブリーフを作成する。
SNS投稿の作成・スケジュール設定は行わない（承認後に design-tool-handoff-jp へ連携）。

```
オーナー: 「今月何を投稿すればいい？」
→ 売上データの取得・分析
→ 売れ筋・鈍足商品の特定
→ 季節性・トレンドの把握
→ 30日コンテンツブリーフの作成
→ オーナー承認
```

## Workflow

### Step 1: データソースの確認

freee、マネーフォワードクラウド、Stripe、または Square のうち接続されているものを確認する。どれも接続されていない場合は売上データを貼り付けてもらう。

### Step 2: 優先指標の確認

オーナーに質問する：
- **「売上の良い商品の基準は何ですか？」**
  - 売上総額？
  - 利益率？
  - 販売速度（どれだけ早く売れるか）？

- **「季節性は把握していますか？」**
  - はいの場合: 教えてもらい、ブリーフに反映する
  - いいえの場合: 業種別ベンチマークを使用する

### Step 3: 売上データの取得・分析

直近90日の売上データを取得する：
- 商品・サービス別の売上と販売数量
- 前期比（可能な場合）
- Stripe の場合: `list_transactions` で取引履歴を取得

分類する：
- **売れ筋（上位3〜5品）**: 選択した指標でトップのもの
- **鈍足（下位3〜5品）**: 売上シェアが低下しているもの
- **上昇トレンド**: 直近30日で伸びているもの
- **下降トレンド**: 直近30日で落ちているもの

### Step 4: 季節性の考慮

- オーナーが季節性パターンを知っている場合: それをブリーフに反映する
- 不明な場合: 業種ベンチマーク（例: 年末商戦、確定申告シーズン等）を参照する
- 来月に向けたポジショニングが必要な商品をフラグする

### Step 5: 30日コンテンツブリーフの作成

以下の構成でまとめる：
- **エグゼクティブサマリー**（1〜2文: 「売れ筋はXとY。ZはZ季節に向けて準備が必要」）
- **積極的に押す**（上位2〜3品 + 推奨コンテンツアングル）
- **維持する**（中位品: 露出は保ちつつ大きな投資は不要）
- **見直す・止める**（鈍足品: 値下げ・バンドル・一時停止を検討）
- **季節の機会**（来月以降の準備が必要なもの）
- **推奨プロモ**（データに基づくセット販売・割引・無料提供案）

### Step 6: オーナーの承認

ブリーフを提示して確認する：
- 「この分析は感覚と合っていますか？」
- 「修正したい点はありますか？」
- 「投稿素材の作成（design-tool-handoff-jp）に進みますか？」

## Approval gates

- **SNS投稿の自動スケジュール設定は行わない。**
- **外部ツールへのデータ書き込みは行わない。**
- **ブリーフはオーナー承認後に次のスキルへ連携する。**
